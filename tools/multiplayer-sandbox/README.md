# Песочница: несколько игроков в браузере

**Не трогает** основной код `app/` и `server/`. Это отдельная папка со своим `package.json`.

## Идея

В веб-режиме (не Telegram) игрок задаётся через `localStorage` (`gameHub_webPlayerId`, `gameHub_webDisplayName` — см. `app/src/useTelegram.js`).

Скрипт открывает **несколько окон Chromium** (каждое — **отдельный контекст** с **своим** `localStorage`), подставляет разные имена/id, чтобы ты видел **разные экраны** одновременно.

## Подготовка (один раз)

Из корня репозитория или из этой папки:

```bash
cd tools/multiplayer-sandbox
npm install
npm run browsers
```

(`browsers` скачает Chromium для Playwright.)

## Как пользоваться

1. Запусти **сервер** и **фронт** как обычно (например `http://localhost:5173`).
2. В другом терминале:

```bash
cd tools/multiplayer-sandbox
npm start
```

По умолчанию откроются **3** окна: Денис, Настя, Игрок 3.

3. В **одном** окне создай комнату, в остальных зайди по приглашению / коду / ссылке.
4. Запусти игру (Элиас и т.д.) и смотри, что у кого на экране.
5. В терминале нажми **Enter** — все окна закроются.

## Настройки (опционально)

| Переменная | Пример | Значение |
|------------|--------|----------|
| `APP_URL` | `http://localhost:5173/lobby` | Стартовая страница |
| `COUNT` | `4` | Сколько окон (1–8), если не задан `PLAYERS_JSON` |
| `PLAYERS_JSON` | см. ниже | Свой список игроков |

```bash
set APP_URL=http://localhost:5173/lobby
set COUNT=2
npm start
```

Свой список (Windows PowerShell):

```powershell
$env:PLAYERS_JSON='[{"id":"web_a","name":"Алиса"},{"id":"web_b","name":"Боб"}]'
npm start
```

## Ограничения

- Нужен установленный Node.js.
- Первый запуск скачивает Chromium (~занимает место).
- Это **не** автотесты с проверками — только удобное открытие нескольких «личностей».

## Если что-то пошло не так

- **Окна Chromium открылись и сразу закрылись**
  1. Сначала в **отдельном** терминале запусти проект: из корня `GAME_HUB` выполни `npm run dev` (нужны и API, и Vite на `http://localhost:5173`).
  2. Запускай песочницу **из обычного терминала** (PowerShell или cmd), а не двойным щелчком по файлу: в неинтерактивном режиме старые версии скрипта сразу завершались и закрывали браузер. Сейчас скрипт ждёт **Ctrl+C**, если Enter недоступен.
- Убедись, что фронт реально на том же хосте/порту, что в `APP_URL`.
- Если навигация падает с ошибкой — в консоли будет текст; чаще всего **не запущен** `npm run dev`.
- Закрой окна вручную и нажми **Enter** в терминале (или **Ctrl+C**, если так написано).
- При ошибке `playwright` — снова `npm run browsers`.
