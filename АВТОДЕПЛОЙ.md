# Как обновлять сервер без ручного захода

## Вариант 1: Git + одна команда на сервере

Код хранится в **Git** (GitHub / GitLab / свой репозиторий). На сервере уже лежит папка проекта — делаем из неё клон репозитория и дальше обновляемся через `git pull`.

### Один раз настроить

1. **Создай репозиторий** на GitHub (или другом сервисе). Не добавляй в репозиторий файл `.env` — он уже в `.gitignore`.

2. **На своём ПК** в папке GAME_HUB выполни (подставь свой URL репозитория):
   ```bash
   git init
   git add .
   git commit -m "first"
   git remote add origin https://github.com/ТВОЙ_ЛОГИН/game-hub.git
   git push -u origin main
   ```
   Если ветка называется `master`, замени `main` на `master`.

3. **На сервере** переименуй текущую папку и клонируй заново (чтобы не потерять `.env`):
   ```bash
   cd /
   mv GAME_HUB GAME_HUB_backup
   git clone https://github.com/ТВОЙ_ЛОГИН/game-hub.git GAME_HUB
   cp GAME_HUB_backup/.env GAME_HUB/.env
   cd GAME_HUB
   docker compose build
   docker compose up -d
   ```
   Потом туннель (cloudflared) снова запусти вручную, если он был в отдельной сессии.

### Каждое обновление

- **На ПК:** правь код, потом:
  ```bash
  git add .
  git commit -m "описание изменений"
  git push
  ```
- **На сервере:** зайти по SSH и выполнить **одну команду**:
  ```bash
  cd /GAME_HUB && bash deploy.sh
  ```
  Либо вручную:
  ```bash
  cd /GAME_HUB && git pull && docker compose build && docker compose up -d
  ```

Файл `.env` на сервере не трогается при `git pull` (его нет в репозитории).

## Вариант 2: Деплой с ПК одной командой (без входа в MobaXterm)

Ты запускаешь скрипт **на своём компьютере** — он по SSH заходит на сервер и выполняет `git pull` и перезапуск контейнеров.

### Один раз настроить

1. **Git** — как в варианте 1 (репозиторий создан, на сервере проект клонирован и уже обновлялся через `git pull` хотя бы раз).

2. **OpenSSH на Windows:** Параметры → Приложения → Дополнительные компоненты → включи **«Клиент OpenSSH»**.

3. **Проверка:** открой PowerShell и выполни:
   ```powershell
   ssh root@5.129.226.192
   ```
   Введи пароль от сервера. Если зашёл — выйди (`exit`).

4. В папке GAME_HUB открой `deploy-from-pc.ps1` и проверь переменные вверху: `$SERVER = "5.129.226.192"`, `$USER = "root"`, `$REMOTE_DIR = "/GAME_HUB"`. При необходимости поменяй.

### Каждое обновление

На ПК в папке GAME_HUB:

1. Сохрани изменения в коде.
2. В PowerShell:
   ```powershell
   git add .
   git commit -m "обновление"
   git push
   .\deploy-from-pc.ps1
   ```
   При запросе введи пароль от root на сервере. Скрипт сам выполнит на сервере `git pull`, сборку и `docker compose up -d`.

Так ты не заходишь в MobaXterm вручную — только запускаешь одну команду с ПК.

## Вариант 3: Автодеплой по push (GitHub Actions)

При каждом `git push` сервер сам подтягивает код и перезапускает контейнеры. Настраивается один раз, дальше достаточно пушить с ПК.

Этот вариант описан отдельно (нужны секреты: хост, пользователь, SSH-ключ). Если захочешь — напиши, распишу по шагам.
