# Motion Guide (GameHub)

## Цели
- Быстрые и понятные переходы без визуального шума.
- Поддержка low-end устройств и `prefers-reduced-motion`.
- Единые длительности для модалок, смены состояний и skeleton.

## Токены
- `--gh-motion-fast`: 140ms (hover/press/focus)
- `--gh-motion-normal`: 220ms (карточки, fade-in, route fallback)
- `--gh-motion-slow`: 320ms (крупные оверлеи)
- `--gh-motion-ease`: `cubic-bezier(0.2, 0, 0, 1)`

## Правила
- Не анимировать больше 2 крупных областей одновременно.
- Для модалок использовать fade/scale, без резких слайдов.
- Таймеры и системные плашки обновлять без дерганий layout.
- Skeleton должен исчезать сразу при готовности данных.

## Reduced Motion
- При `prefers-reduced-motion: reduce` анимации отключаются (1ms).
- Критичные статусы остаются видимыми через цвет/текст, а не через движение.

## QA-проверка
- Открытие/закрытие модалки: плавно, без прыжка фокуса.
- Переключение страниц: виден skeleton/fallback, нет белого экрана.
- На слабом устройстве интерфейс не «тормозит» от анимаций.
