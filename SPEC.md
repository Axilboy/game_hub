# ТЗ: Игровой хаб (Telegram Mini App)

## 1. Общая концепция

**Продукт:** Мини-приложение в Telegram для совместных игр в одной комнате (по типу Jackbox), где **каждый участник использует только свой телефон**. Приложение — ассистент: показывает роли, слова, таймеры, голосования.

**Первый релиз:** только игра «Шпион». Дальше — Мафия, Элиас, Бункер и др.

---

## 2. Роли и сценарии

### 2.1 Хост
- Запускает приложение и создаёт комнату.
- Получает **приглашение (invite)** для комнаты (например кнопка «Пригласить игроков»), чтобы отправить его в чат/друзьям.
- Выбирает игру из списка (пока только «Шпион»).
- Настраивает параметры игры (если есть).
- Запускает игру, когда все зашли.
- Может видеть состояние игры (кто в комнате, готовность и т.п.).

### 2.2 Игрок
- Переходит в комнату **по приглашению от хоста** (например по ссылке/кнопке из чата).
- Видит название выбранной игры и ожидание старта.
- После старта получает свою «карточку»: слово (или уведомление «Вы шпион»).
- Использует приложение только как подсказку; обсуждение — голосом в комнате.

### 2.3 Ограничения
- Один хост на комнату.
- Игрок не может быть в двух комнатах одновременно (по текущему Telegram user).
- Приглашение в комнату (invite) действительно, пока комната не закрыта или не истекло время жизни (если решим делать TTL).

## 3. Игра «Шпион» (первая реализация)

### 3.1 Правила (кратко)
- Всем раздаётся одно и то же **слово** (локация/профессия/объект), кроме одного или нескольких **шпионов**.
- Шпион видит только сообщение вроде: «Вы шпион» (слово не знает).
- Игроки задают друг другу вопросы по слову, шпион должен не выдать себя.
- Цель мирных — вычислить шпиона; цель шпиона — угадать слово или остаться незамеченным.

### 3.2 Что делает приложение
- Хост: выбор игры «Шпион», при необходимости настройки (кол-во шпионов, категория слов).
- После старта каждому игроку показывается:
  - либо **слово** (одно и то же у не-шпионов);
  - либо текст **«Вы шпион»** (и без слова).
- Слова берутся из заранее подготовленного набора (файл/БД). Категории по желанию: локации, профессии, предметы и т.д.

### 3.3 Раунд
- Один раунд = одна раздача слов + один шпион(ы).
- После раунда хост может начать новый раунд или завершить игру и вернуться в лобби.

### 3.4 Что НЕ делаем в первой версии (можно вынести в «потом»)
- Таймер раунда.
- Голосование «кто шпион» в приложении (пока только устно).
- Рейтинги и статистика.

## 4. Будущие игры (только перечень для ТЗ)

- **Мафия** — роли (мафия, мирные, комиссар и т.д.), ночь/день, голосования.
- **Элиас** — команды, одно объясняет слово, другие угадывают, таймер.
- **Бункер** — роли и условия выживания, обсуждение и голосование.

Механики и экраны для них описываются отдельными дополнениями к ТЗ.

## 5. Технические требования

### 5.1 Платформы
- **Telegram Mini App** (Web App), открывается из бота или меню.
- Адаптивная вёрстка: в первую очередь **телефон** (хост и игроки).

### 5.2 Backend
- Сервер с API (Node.js / Python / другой — на твой выбор).
- **Реальное время:** игроки в одной комнате должны почти сразу видеть изменения (кто зашёл, игра началась, раздача слов). Варианты: WebSockets (Socket.io и т.п.) или long polling. Рекомендация: WebSockets.
- Хранение: комнаты, активные игры, инвайты. Достаточно in-memory на первом этапе с возможностью позже перенести на Redis/БД.

### 5.3 Комната
- **Invite-токен** комнаты генерируется сервером и используется для присоединения по приглашению.
- В комнате: `host_id`, список `players[]`, выбранная игра, состояние (ожидание / игра идёт), параметры игры.
- При создании комнаты хост получает invite и остаётся в ней до закрытия или выхода.

### 5.4 Безопасность и идентификация
- Идентификация через **Telegram Web App** (initData / user от Telegram). Не храним пароли; достаточно `user_id` (или хэша) для привязки к комнате и ролям.
- Присоединение в комнату — **по приглашению от хоста** (invite-ссылка/кнопка), чтобы контролировать доступ и упростить будущую монетизацию.

### 5.5 Словарь для «Шпион»
- Набор слов (и при желании категорий) хранится на боте/сервере. Формат: например JSON или таблица в БД. При старте раунда сервер выбирает случайное слово и случайных шпионов среди игроков в комнате.

## 6. Экраны (кратко)

| Кто    | Экраны |
|--------|--------|
| Все    | Старт → «Создать комнату» ИЛИ «Войти по приглашению». |
| Хост   | Лобби (кнопка «Пригласить игроков», список игроков, выбор игры «Шпион», кнопка «Начать»). Настройки Шпиона (если есть). После старта — экран «Игра идёт» (можно минимальный: «Раунд начался» + кнопка «Следующий раунд» / «Завершить»). |
| Игрок  | Лобби (ожидание старта). После старта — экран с **словом** или с **«Вы шпион»**. По желанию кнопка «Показать снова» в рамках раунда. |

## 7. Что уточнить / решить

### Принятые решения
- Стек: **выбирается исполнителем** (приоритет — качество и красивый UX).
- Клиенты: **Telegram Mini App** как первый клиент; архитектура с прицелом на **будущие Android/iOS приложения**, которые смогут подключаться к тем же комнатам.
- Вход в комнату: **по приглашению-ссылке** и **по коду** (оба варианта поддерживаются).
- Язык: **только русский** на первом этапе.
- Лимит игроков: **пока не ограничиваем** (можно ввести позже, если понадобится).
- Хостинг/деплой: будет **отдельный сервер**, настройка/деплой входят в работы.
- Монетизация: будет добавляться позже; в первом релизе закладываем только совместимую основу (invite-only/код, контроль доступа).

## 8. Этапы разработки (черновик)

1. **Каркас:** бот + мини-приложение (открытие, базовая вёрстка), бэк с WebSockets, создание комнаты и invite, присоединение по приглашению.
2. **Лобби:** список игроков в реальном времени, выбор «Шпион», кнопка «Начать».
3. **Шпион:** словарь слов, раздача слова/шпион, экраны для хоста и игроков, переход «следующий раунд» / «в лобби».
4. **Полировка:** обработка отключений, переподключение, подсказки в UI.
5. Дальше: Мафия, Элиас, Бункер по отдельным ТЗ.
